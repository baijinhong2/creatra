# v0.7.1 — Onboarding UX 重设计(变更留档)

> **状态**: 设计定稿,开发中
> **作者**: Mavis
> **基于**: `v0.7-product-optimization.md` §4.1.1 + 用户反馈(2026-07-16 凌晨)
> **变更范围**: Onboarding 路径入口 + Path 1/2/3/4 全部重做
> **目标**: 让首次设置 voice DNA 的人,在 30 秒 - 10 分钟内走完,且真的像他

---

## 1. 背景 / 为什么改

### v0.7 实际跑出来的问题

跑 v0.7 spec 的 Onboarding 后,用户(产品作者)走了一遍,发现 3 个具体问题:

| # | 问题 | 原因 |
|---|---|---|
| 1 | "直接选模板"藏在 Path B 下面,新人不容易找到 | 3 路径分类错了维度 |
| 2 | Path B 进去还要选 "粘 / 推荐 / 模板" 3 选 1 | 入口太多,模板根本不属于这层 |
| 3 | Path C (我没X账号) 5 题 quiz 表面,没真了解用户 | quiz 形式适合 fast matching,不适合 deep understanding |

### 设计原则调整

- **从"3 类用户" → "4 类用户"**:增加"我想跟某博主"独立路径,把模板提到第一级
- **从"粘推文 → 拉数据 → 提取" → "多源内容"**:Path 1 加粘文字子选项(无 cookies 也能用)
- **从"quiz 多选题" → "对话式多选 + 自由输入"**:Path 4 重做成 6 题对话

---

## 2. 新结构(4 路径入口)

```
先认识你的写作风格
  │
  ├─ [1] 🎯 我有内容                   拉我的推文/博客/Newsletter → 10-30s
  │     └─ 1a: 粘 X handle              (拉 25 条推文,需要 cookies)
  │     └─ 1b: 粘文字                   (粘贴自己的内容,不需要 cookies) ⭐ 新增
  │
  ├─ [2] 🧬 直接选模板 ⭐ 推荐          5 个 voice 模板,选 1 个就走 → 30s
  │
  ├─ [3] 📋 我想用某博主的风格          粘对标 handle → 1-2 min
  │     └─ 3b: "不知道对标谁?让我推荐"  (3 题 quiz → 5 个推荐 → 回 3)
  │
  └─ [4] ✨ 从 0 开始 / 聊聊自己         6 个问题 + 自由输入,AI 合成 → 5-10 min
```

**4 个路径的对比维度**:

| 路径 | 用户已有 | 速度 | X cookies | LLM 调用 |
|---|---|---|---|---|
| 1a 我有内容 - handle | 自己的 X 推文 | 10s | 必需 | 1 次提取 |
| 1b 我有内容 - 文字 | 任何文字 | 20s | 不需要 | 1 次拆分 + 1 次提取 |
| 2 直接选模板 | 无 | 30s | 不需要 | 0(配置) |
| 3 用某博主的风格 | 别人的推文 | 1-2 min | 必需 | 1 次提取 |
| 4 从 0 开始 | 无 | 5-10 min | 不需要 | 1 次合成 |

## 3. 每个 Path 的详细设计

### Path 1a: 我有内容 - 粘 X handle

```
Step 1: 粘 handle
  - 单一 input 框,占主要位置
  - 大字: "粘你的 X handle"
  - 描述: "我会拉你最近 25 条原创推文,提取你的风格"
  - 输入框: [@_____________]
  - [提取 DNA →] 按钮
  - 边界提示: "需要 X cookies 配好;没有的话用 1b 粘文字"

Step 2: 提取结果(与 1b / 2 / 3 共用 result page)
  - DNA 特征卡片
  - 3 条代表推文
  - [✓ 用这个] [调整] [重新提取]
```

### Path 1b: 我有内容 - 粘文字 ⭐ 新增

```
Step 1: 粘文字
  - 大字: "粘你自己的内容"
  - 描述: "粘贴你最近写的 5-10 条推文、博客段落、newsletter 段落、或邮件开头"
  - 来源标签(可多选):
    □ X 推文  □ 博客  □ Newsletter  □ 邮件  □ 其他
  - 大 textarea(rows=12, 200-5000 字):
    [________________________________]
    [________________________________]
    [________________________________]
  - 字数统计: 245 / 5000 字
  - [提取 DNA →] 按钮

Step 2: 服务端处理
  - POST /api/voice-dna/extract-from-text
  - LLM 拆分成 ≥ 5 个独立片段(每段 ≥ 10 字符)
  - 调同一 dnaExtraction prompt(像 Path A 一样)
  - 失败时: "看起来内容太单一,试试粘贴多段"

Step 3: 共享 result page
```

**为什么这个重要**:
- 没 cookies 也能用(避免用户卡在配置步骤)
- 博客/newsletter 用户能直接用(不是只支持 X 推文)
- 数据多样性更好(博客 = 长篇,推文 = 短句,组合后 DNA 更准)

### Path 2: 直接选模板

```
1. 5 个 voice 模板横排(原 Path B3 上提)
   - Build-in-Public Operator
   - Hot Take Provocateur
   - Patient Teacher
   - Storyteller Confessional
   - Industry Curator
2. 点开看 3 条样例
3. 选 1 → DNA 立即套用
4. 存库

特点:
- 0 输入,30 秒完成
- 不需要 cookies
- 适合"我想试试水"的新用户
- "推荐" 标签 — 最低门槛
```

### Path 3: 我想用某博主的风格

```
Step 1: 粘对标 handle (主要路径)
  ┌─────────────────────────────────────┐
  │ 粘 1-3 个你欣赏的 X 博主            │
  │                                      │
  │ 1. [_______________]                 │
  │ 2. [_______________]                 │
  │ 3. [_______________]                 │
  │                                      │
  │ [提取 DNA →]                         │
  │                                      │
  │ ─────────────────────────            │
  │ 不知道对标谁?                       │
  │ [让我帮你推荐 →]   ← 次级入口       │
  └─────────────────────────────────────┘

Step 1b (按了"让我推荐"):
  - Q1: 方向(SaaS/AI/工具/内容/其他)
  - Q2: 风格偏好(casual/专业/犀利/温暖)
  - Q3: 语言(中/英)
  - [推荐]
  → 显示 5 个候选 handle(从 web_search 抓)
  → 选 1-2 → 回到 Step 1(自动填入 input)
  → 继续 Step 2

Step 2: 提取 + 展示
  - 与 Path 1 共用 result page
```

**对比 v0.7 的差异**:
- ❌ v0.7:Path B 进去先选 "粘 / 推荐 / 模板" 3 选 1(模板隐藏)
- ✅ v0.7.1:进来直接粘 handle,推荐作为补充(模板独立到 Path 2)

### Path 4: 从 0 开始 / 聊聊自己(大改)

**核心设计原则**:**不是 quiz,是一次对话**。每个问题给选项降低门槛,但**始终允许自由输入**让用户补充。

```
Step 1/6: 你现在主要在做什么?
  选项(可多选):
    □ 独立开发  □ 工程师  □ 设计师
    □ 产品经理  □ 学生    □ 创作者
    □ 创业者    □ 其他
  自由输入(1 句话补充):
    [________________________________________]
    例: "我是前端工程师,业余做独立 SaaS"

Step 2/6: 你想在 X 上聊什么?
  选项(可多选): 技术细节 / 产品复盘 / 行业观察 /
              个人思考 / 教程 / 故事 / 其他
  自由输入:
    [________________________________________]
    例: "AI agent、RAG 实践、独立开发踩坑"

Step 3/6: 你希望从 X 获得什么?
  选项(可多选): 影响力 / 技术人脉 / 客户或收入 /
              单纯记录 / 求职 / 行业地位
  自由输入:
    [________________________________________]

Step 4/6: 你的表达风格?
  长度(单选): 短句 / 中段 / 长段
  语气(单选): 严肃 / 轻松 / 犀利 / 温暖
  emoji(单选): 0 / 偶尔 / 经常
  自由输入:
    [________________________________________]

Step 5/6: 你欣赏什么样的写作者?(可选)
  自由输入(多行):
    [________________________________________]
    [________________________________________]
    提示:可以是人名,可以是一句话,
         可以是"我希望像跟朋友聊天一样"
  [跳过此步] [下一步 →]

Step 6/6: 如果现在发一条 X 推文,你会发什么?(可选)
  自由输入:
    [________________________________________]
  参考 3 个 voice 模板的样例(可点开看):
    [📋 Operator] [📋 Teacher] [📋 Storyteller]
  [跳过此步] [下一步 →]

Step 7: DNA 预览
  - 基于你 6 个回答 + 自由输入合成的 DNA
  - 3 条代表样例
    - Q6 写了的用你的(像 Path A 那样分析)
    - 没写用模板默认
  - 合成理由(我为什么这么写)
  - [✓ 用这个 DNA] [调整] [回到上一步修改]
```

**每步都可跳,总时间 5-10 分钟**(快速版本 Q1-Q4 仅 2-3 分钟)。

**单页 + 内部 state**:
- 不用 URL params(localStorage 备份进度)
- 一页搞定 6 步,平滑切换
- 刷新页面从断点继续

## 4. DNA 合成逻辑(关键变化)

### Path 1a/3 (handle 拉推文) - 不变
原 spec,无变化。

### Path 1b (粘文字) - 新增

```ts
// Step 1: LLM 拆分
const SPLIT_PROMPT = `把以下用户内容拆分成 5-10 个独立片段(段落)。
每个片段应该:
- 至少 10 字符
- 是完整的句子或段落
- 不重叠
- 反映不同的写作场景(开头/中间/结尾)

输入:
"""
{USER_TEXT}
"""

输出 JSON:
{"chunks": ["...", "...", ...]}

只返回 JSON。`;

// Step 2: 拆完后调同一 dnaExtraction prompt
// 与 Path A 完全相同
```

### Path 2 (template) - 不变
直接写入模板的 DNA,confidence 0.5。

### Path 4 (对话) - 新增

```ts
const CONVERSATION_SYNTHESIS_PROMPT = `基于用户 6 个回答 + 自由输入,合成 voice DNA。

## 用户输入
- Q1 做什么: ${q1_activities.join(',')}, "${q1_text}"
- Q2 聊什么: ${q2_topics.join(',')}, "${q2_text}"
- Q3 获得什么: ${q3_goals.join(',')}, "${q3_text}"
- Q4 风格: 长度=${q4_length}, 语气=${q4_tone}, emoji=${q4_emoji}, "${q4_text}"
- Q5 欣赏: ${q5_text || '(跳过)'}
- Q6 写一条: ${q6_text || '(跳过)'}

## 合成规则
1. **Q6 有内容**:把它当主样本(像 Path A 那样提取 DNA)
2. **Q5 详细**:用具体人/风格匹配 template,叠加
3. **Q1-Q4 调整**:
   - Q1 方向 → topics
   - Q2 聊什么 → topics, signature_patterns
   - Q3 获得什么 → topics, signature_patterns
   - Q4 长度/语气/emoji → structure, tone, emoji_rate
4. **所有 free text** → 补充 signature_patterns
5. **confidence**:
   - Q6 + Q5: 0.7
   - Q6 仅有: 0.6
   - Q5 详细: 0.5
   - 只 Q1-Q4: 0.4
6. **不编造** — 样本偏少就标,不要凑数

## 输出
完整 voice DNA JSON(同其他 path schema)。`;
```

## 6. 新增 API

```
POST /api/voice-dna/extract-from-text
  Body: { text: string, source_types: string[] }
  → 1) LLM 拆 5+ 片段  2) 同一 dnaExtraction prompt  3) 存库
  → 返回 { success, dna }

POST /api/voice-dna/synthesize-from-conversation
  Body: { q1, q2, q3, q4, q5?, q6? }
  → 1) LLM 合成 DNA  2) 存库
  → 返回 { success, dna }
```

## 8. 边界 / Edge Cases

| 情况 | 处理 |
|---|---|
| Path 1a 无 cookies | "需要 X cookies,或用 1b 粘文字" |
| Path 1a 拉 0 条 | "账号新 / 推文少,试试 1b 粘文字 或 Path 2 选模板" |
| Path 1b 文字 < 200 字 | "至少 200 字,让 AI 更好识别你的风格" |
| Path 1b 拆不出 5+ 段 | "内容太单一,加几段其他类型的(开头/中间/结尾)" |
| Path 1b 总长度 > 5000 字 | "太长了,先截取 5-10 段有代表性的" |
| Path 3 handle 不存在 | "账号找不到或私密,换 1 个试试 或 让我推荐" |
| Path 3 cookies 失效 | "cookie 失效,先去 Sources 配" |
| Path 4 用户全选"无所谓" | 兜底:中庸默认 DNA(中性 casual balanced) |
| Path 4 用户刷新 | localStorage 恢复进度,从 Q1-N 继续 |
| 用户中途退出 onboarding | localStorage 存 progress + 答案,回来从断点继续 |
| 用户重做 onboarding | 旧 version 不删,新 version +1 |
| Path 4 Q6 用户写了一段对话 | "请写一条 X 推文,而不是聊天" (提示) |
