# v0.7.2 — UI/UX 全面统一重构

> **范围:** 全站(15+ 页面/组件)UI/UX 清理
> **目标:** 单一 design system,统一字号/配色/icon/文案,提升专业感
> **预计工作量:** 6-7 小时
> **不做:** 新功能、性能优化、a11y 深度改造、暗色以外的主题

---

## 1. 背景与目标

### 1.1 为什么做

经过 v0.7 + v0.7.1 之后,功能层基本完整。但产品存在**视觉与文案一致性问题**,用户视角下感受:
- "界面整体很乱,不知道重点"
- "有的字号很小有的很大"
- "中英文混着"
- "emoji 当 icon 看起来不专业"
- "文案太多"

### 1.2 根因(已通过代码审计 + 截图比对确认)

- **没有 design system:** 字号、配色、icon 各自为政
- **Emoji 当 icon 用:** 83 处,43 种,11 文件 —— 用户明确不要
- **字号双轨制:** Tailwind `text-xs/sm/base/...` 与 arbitrary `text-[Npx]` 并存,17 个不同尺寸
- **中英混杂:** 无明确规则,中英随机出现
- **文案密度过高:** 多页有冗余文字(重复说明、副标题过长、占位符冗长)

### 1.3 目标

1. 全站**只用 6 档字号**(`text-xs/sm/base/lg/xl/2xl`),禁 `text-[Npx]`
2. 全站**只用 10 个语义色 token**,禁裸 `bg-zinc-X` / `bg-violet-X`
3. 全站**零 emoji**,所有 icon 走 `lucide-react`
4. 全站**中英规则明确**:zh 界面以中文为主,仅保留 3 类英文(产品名/技术专有名词/已通用术语)
5. 全站**按钮规则明确**:A 类通用操作 → icon-only;B 类功能动作 → icon + 短词;C 类标签 → 纯文字
6. 全站**文案密度**收敛:每个页面的"重点一眼可见"

### 1.4 非目标

- 不做暗色以外的 theme 切换
- 不动业务逻辑
- 不重写 agent / 工具调用
- 不重写 API 路由

### 3.3 `<Text>` 组件(`src/components/Text.tsx`)

```tsx
import { FONT_SIZE, COLOR, type FontSize, type ColorToken } from '@/lib/design';
import type { ReactNode } from 'react';

type Props = {
  size?: FontSize;
  tone?: 'text' | 'muted' | 'subtle' | 'danger' | 'success' | 'warn';
  weight?: 'normal' | 'medium' | 'semibold' | 'bold';
  as?: 'p' | 'span' | 'div' | 'label';
  children: ReactNode;
  className?: string;
};

const WEIGHT_CLASS: Record<NonNullable<Props['weight']>, string> = {
  normal:   'font-normal',
  medium:   'font-medium',
  semibold: 'font-semibold',
  bold:     'font-bold',
};

export function Text({ size = 'small', tone = 'text', weight, as: As = 'p', children, className = '' }: Props) {
  const weightClass = weight ? WEIGHT_CLASS[weight] : '';
  return (
    <As className={`${FONT_SIZE[size]} ${COLOR[tone as ColorToken].light} ${COLOR[tone as ColorToken].dark} ${weightClass} ${className}`}>
      {children}
    </As>
  );
}
```

### 3.4 `<Button>` 组件(`src/components/Button.tsx`)

```tsx
import { ICON, RADIUS, type IconName } from '@/lib/design';
import type { ReactNode } from 'react';

type Variant = 'primary' | 'secondary' | 'ghost' | 'danger';

type Props = {
  variant?: Variant;
  icon?: IconName;            // 主 icon
  trailingIcon?: IconName;    // 末尾 icon(可选)
  iconOnly?: boolean;         // A 类:icon-only,正方形
  size?: 'sm' | 'md';
  children?: ReactNode;
  onClick?: () => void;
  disabled?: boolean;
  typeAttr?: 'button' | 'submit';
  className?: string;
  title?: string;
};

export function Button({
  variant = 'secondary', icon, trailingIcon, iconOnly,
  size = 'md', children, onClick, disabled, typeAttr = 'button', className = '', title,
}: Props) {
  const IconComp = icon ? ICON[icon] : null;
  const TrailingComp = trailingIcon ? ICON[trailingIcon] : null;
  const sizeClasses = size === 'sm' ? 'h-8 text-xs px-2.5' : 'h-10 text-sm px-4';
  const iconSize = size === 'sm' ? 14 : 16;

  const variantClasses = {
    primary:   'bg-violet-600 hover:bg-violet-700 text-white',
    secondary: 'border border-zinc-200 dark:border-zinc-700 bg-white dark:bg-zinc-900 text-zinc-900 dark:text-zinc-50 hover:bg-zinc-50 dark:hover:bg-zinc-800',
    ghost:     'text-zinc-600 dark:text-zinc-400 hover:bg-zinc-100 dark:hover:bg-zinc-800',
    danger:    'text-red-700 dark:text-red-300 hover:bg-red-50 dark:hover:bg-red-950/30',
  }[variant];

  const baseClasses = 'inline-flex items-center justify-center font-medium transition disabled:opacity-50 disabled:cursor-not-allowed';
  const shapeClasses = iconOnly ? 'w-10 h-10' : `${sizeClasses} gap-1.5 ${RADIUS.sm}`;

  return (
    <button
      type={typeAttr}
      onClick={onClick}
      disabled={disabled}
      title={title}
      className={`${baseClasses} ${shapeClasses} ${variantClasses} ${className}`}
    >
      {IconComp && <IconComp size={iconSize} strokeWidth={2} />}
      {!iconOnly && children}
      {TrailingComp && !iconOnly && <TrailingComp size={iconSize} strokeWidth={2} />}
    </button>
  );
}
```

### 3.5 `<Card>` 组件(`src/components/Card.tsx`)

```tsx
import { RADIUS } from '@/lib/design';
import type { ReactNode } from 'react';

type Props = {
  children: ReactNode;
  className?: string;
  highlight?: boolean;  // 主推/高亮卡片,加紫色边框
  padding?: 'sm' | 'md' | 'lg';
};

export function Card({ children, className = '', highlight = false, padding = 'md' }: Props) {
  const padClass = { sm: 'p-3', md: 'p-4', lg: 'p-5' }[padding];
  const baseClass = `${RADIUS.lg} border bg-white dark:bg-zinc-900 ${padClass}`;
  const borderClass = highlight
    ? 'border-violet-300 dark:border-violet-800'
    : 'border-zinc-200 dark:border-zinc-800';
  return <div className={`${baseClass} ${borderClass} ${className}`}>{children}</div>;
}
```

## 4. 文案规则

### 4.1 语言规则(中英混杂处理)

**保留英文的 3 类:**
1. **产品名/品牌名**:creatra(域名 creatra.xyz),agent 自我称呼 社交运营顾问(Social Advisor),DeepSeek, X
2. **技术专有名词**(用户已熟悉):handle, retweet, thread, API, cookie
3. **当前 dev 上下文专用术语**:Voice DNA, Inbox 收件箱(作为模块名可保留 Inbox)

**必须翻译的:**
- 互动 Inbox → 互动收件箱
- score → 匹配度
- Voice DNA(用于产品名 OK,作为技术概念时)→ 声音 DNA
- trends → 热点/趋势
- outcomes → 历史表现
- force, sync, draft → 翻译

### 4.2 密度规则

每个页面的"重点"应该在第一眼可见:
- 标题 ≤ 12 字
- 副标题 ≤ 30 字
- 提示性文字 ≤ 1 行
- placeholder ≤ 20 字
- 按钮文案 ≤ 4 字(中文)
- section 标签 ≤ 6 字

**自动隐藏**(默认折叠,需要时展开):
- reasoning / 为什么推荐 / 详细分析
- 备选 / 备选方案
- 历史记录
- 详细说明文字(用 tooltip / details 替代)

### 4.3 Unicode 符号也要替换

以下不是严格意义上的 emoji(Dingbats 区块),但同样不是"标准 icon",要一起替换:

| 现状 | 替换为 |
|---|---|
| `✓` (U+2713) | `[Check]` 或 `[CheckCircle2]` |
| `✕` (U+2715) | `[X]` |
| `⚙` (U+2699) | `[Settings]` |
| `⚠` (U+26A0) | `[AlertTriangle]` |
| `☀` `☾` (U+2600 U+263E) | `[Sun]` `[Moon]` |
| 国旗 emoji (🇺🇸 🇨🇳 等) | 纯文字 "English" / "中文" |

### 4.4 按钮规则(3 类)

| 类型 | 何时用 | 视觉 | 例子 |
|---|---|---|---|
| **A. 通用操作** | 用户一看 icon 就知道(复制/同步/关闭/折叠/删除) | icon-only,32×32,hover tooltip | `[Copy] [RefreshCw] [X] [ChevronDown] [Trash2]` |
| **B. 功能动作** | 一次主要操作,需要短文字 | icon + 短词(< 4 字中文) | `[Wand2] 起草回复` · `[RefreshCw] 重新生成` |
| **C. 标签/状态** | 静态 section heading | 纯文字,无 icon | "主推" · "建议" · "总结" |

**icon-only 阈值清单(A 类可用):**
- 复制 Copy / 同步 RefreshCw / 关闭 X / 展开 ChevronDown / 收起 ChevronUp / 删除 Trash2 / 加号 Plus / 减号 Minus / 设置 Settings / 回到顶部 ArrowUp

**B 类必须配文字的场景:**
- "换一条" / "重新生成" / "起草回复" / "保存" / "提取" / "保存并下一步" / "标记已处理"
