# Phase 2 Spec — Topic Intelligence + Account Health

> **Implementation spec for**: Topic Intelligence 选题智能 + Account Health 周报 + 教学 layer
> **Based on**: `需求/v0.7-product-optimization.md` §6 (Phase 2)
> **Status**: ready for implementation
> **Prerequisite**: Phase 0 (Voice DNA + Outcomes) + Phase 1 (Engagement 数据)
> **Last updated**: 2026-07-15
> **Estimated effort**: 1.5-2 周

---

## 1. Goal

从"agent 写"升级到"agent 选题 + 写 + 复盘"。

**三个核心能力**:
1. **Topic Intelligence** — 主动告诉用户"今天可以发什么",从被动变主动
2. **Account Health** — 每周体检,让用户看到自己的账号趋势
3. **Teaching Layer** — 每次开新对话,主动展示"我刚从你账号学到的",强化学习闭环

**依赖**:Topic Intelligence 和 Teaching Layer 都需要 Outcomes Loop 的 pattern 数据(谁的高互动、什么时段有效)。Account Health 需要 metrics 拉取能力(Phase 0 已建)。

### Topic Intelligence

| ID | Story |
|---|---|
| US-1 | 作为 Zhuanz,每天 9 点 agent 主动告诉我"今天有 3 个新热点:['vibe coding' 讨论降温,'AI agent memory' 在升温,'Typefully 新功能' 引发讨论]。其中 #2 跟你最贴(你之前写过 5 篇),建议今天发 'AI agent memory 实战'角度" |

| US-3 | 作为 Zhuanz,推荐附带 1 条主推 + 4 条备选,主推可直接 copy,备选可微调 |
| US-4 | 作为 Zhuanz,如果今天没新热点,agent 不强推,只说"今天没明显新热点,要不要做你账号最有效的'复盘类'?" |

| US-5 | 作为 Zhuanz,每周日晚上收到"📊 你的 X 账号 · 过去 7 天" 报告 |

| US-7 | 历史报告可回顾(过去 4 周都能看),用于对比趋势 |

### Teaching Layer

| US-8 | 作为 Zhuanz,每次开新对话,agent 开头展示 1-2 条"💡 我从你账号学到的" |
| US-9 | 不刷屏,只在合适时机(新对话开头 / 用户停顿 1+ 天回来) |

## 4. API Changes

### 4.1 `POST /api/topics/recommend`

**用途**:生成今日推荐(1 主推 + 4 备选)。

**Request**:
```ts
{
  user_handle: string,
  date: string,                          // 'YYYY-MM-DD',默认 today
  force: boolean,                        // true = 忽略缓存,重生成
}
```

**Response**:
```ts
{
  success: true,
  date: string,
  main_recommendation: {
    topic: string,                       // 'AI agent memory 实战'
    angle: string,                       // '从你之前 5 篇中提炼的'3 个关键卡点''
    draft: string,                       // 完整推文草稿
    reasoning: {
      why_this_topic: string,            // '升温中,你之前写过 5 篇'
      why_this_angle: string,            // '复盘类你互动高 3x'
      voice_dna_match: string,           // '匹配你的 casual 短句风格'
      outcomes_evidence: string,         // '过去 30 天你 3 条类似推文平均互动 28'
    },
    best_time: { start: 'HH:MM', end: 'HH:MM', tz: 'Asia/Shanghai' },
    alternatives: [
      { topic: string, angle: string, draft: string, score: 0-1 },
      ...                                // 共 4 个
    ],
  },
  trends: [
    { topic: string, status: 'rising' | 'peaked' | 'cooling' | 'new', momentum: -1 to 1 },
    ...
  ],
  no_significant_trend: boolean,         // 真的没热点时为 true
}
```

**实现要点**:
1. 拉用户 watchlist 推文 + 近 7 天该赛道 web 搜索 → 找热点
2. 调 `outcomesAnalysis.ts` 拉用户过往 pattern(高互动类型 / 时段)
3. 调 `voiceDna.ts` 拉 voice DNA
4. 调 `topicIntelligence.ts` LLM 推理 → 输出 1 主推 + 4 备选
5. 存到 `vp_topic_recommendations` 表

**频率**:用户手动触发(在 chat 说"今天发什么"或点按钮) + cron 每天 9 点自动生成(可关)。

### 4.4 `POST /api/health/generate`

**用途**:生成周报。

**Request**:
```ts
{
  user_handle: string,
  week_start: string,                    // 'YYYY-MM-DD',默认上周日
}
```

**Response**:
```ts
{
  success: true,
  report: {
    week: string,
    generated_at: timestamptz,
    metrics: {
      followers: { start: int, end: int, delta: int, delta_pct: float },
      tweets_published: int,
      avg_engagement: float,
      best_time_slot: { day: string, hour_range: string, multiplier: float },
      best_type: { type: string, avg_engagement: float, count: int },
      worst_type: { type: string, avg_engagement: float, count: int },
      new_followers_sample: [{ handle, bio, follower_count }, ...],  // 最多 10
      most_quoted: { tweet_text, quote_count, url },                 // 1 条
    },
    outcomes_summary: string,            // '复盘类互动 28,教程类 12,观点类 8'
    recommendations: [string, string, string],  // 3 条具体建议
  }
}
```

**实现要点**:
1. 调 `twitter_get_user_tweets(handle, count=100)` 拉近 30 天推文
2. 调 `twitter_get_tweet_replies(tweet_id)` + `pull_tweet_metrics` 拉每条 metrics
3. 计算各种统计(最佳时段、最佳类型等)
4. 调 outcomes pattern 叠加
5. 调 LLM 写 3 条建议
6. 存到 `vp_health_reports` 表

**频率**:cron 每周日 21 点自动生成(用户时区)。

### 4.6 `GET /api/insights/teaching-layer`

**用途**:新对话开头展示教学 insights。

**Query**:`?user_id=...&conversation_id=...`

**Response**:
```ts
{
  insights: [
    {
      id: string,
      type: 'pattern' | 'tip' | 'warning',
      text: string,                       // '你上周 5 条复盘贴互动平均 28,其他类型 12'
      evidence: string,                   // 数据来源
      dismissable: boolean,
      already_dismissed: boolean,
    }
  ],
  show_count: 1-2,                       // 限制不刷屏
}
```

**实现要点**:
1. 加载用户 outcomes pattern + 健康报告最新一周
2. 选 1-2 条最有价值的(优先 warning > pattern > tip)
3. 检查 `dismissed_insights` 表,过滤已 dismiss 的
4. 用 LLM 把数据转成"自然语言"教学(不要列数据,要"我观察到...")

**触发时机**:
- 新对话开头第一条
- 用户停顿 ≥ 1 天回来(用 `last_message_at` 判断)
- 不在每条消息都触发(避免刷屏)

## 6. LLM Prompts

### 6.1 Topic Intelligence — `recommendTopics(user, dna, outcomes, trends)`

```ts
const TOPIC_INTEL_PROMPT = `你是 X 增长选题专家。基于用户账号数据 + 当前热点,推荐今天最值得发的 1 条主推 + 4 条备选。

## 真实性自审
1. **不编造热点** — 只能基于下面输入的 trends 数据,不能自己编"最近 X 在火"
2. **不重复推已有内容** — 跟用户过往 100 条推文去重
3. **不堆通用建议** — "好内容互动高"不算推荐,必须给具体 topic + angle
4. **必须能落地** — 推荐要带完整 draft 推文,不只是"建议谈 X"

## 输入
### 用户 voice DNA
${JSON.stringify(dna.features)}

### 用户过往 100 条推文标题(去重,按时间倒序)
${recent_tweet_topics.slice(0, 100).map((t, i) => `${i + 1}. ${t}`).join('\n')}

### 用户过往高互动 pattern(从 Outcomes Loop)
${outcomes_patterns.map(p => `- ${p.pattern} (confidence: ${p.confidence})`).join('\n')}

### 当前热点(从 web 搜索 + watchlist 推文流)
${trends.map(t => `- ${t.topic} [${t.status}, momentum: ${t.momentum}]`).join('\n')}

## 输出 JSON
{
  main_recommendation: {
    topic: string,                // 'AI agent memory'
    angle: string,                // '从你卡点 3 周的实战提炼'
    draft: string,                // 完整推文草稿,带 voice DNA
    reasoning: {
      why_this_topic: string,     // '升温中,你之前写过 5 篇相关'
      why_this_angle: string,     // '复盘类你互动高 3x'
      voice_dna_match: string,    // '匹配 casual 短句'
      outcomes_evidence: string,  // '过去 30 天你 3 条类似推文平均互动 28'
    },
    best_time: { start: 'HH:MM', end: 'HH:MM', tz: 'Asia/Shanghai' }
  },
  alternatives: [                 // 4 个备选,结构同 main 但可更短
    { topic, angle, draft, score: 0-1 }
  ],
  trends: [                       // 复用输入
    { topic, status, momentum }
  ],
  no_significant_trend: boolean
}

如果 trends 都是 cooling/peaked 且匹配用户过往内容都写过,设 no_significant_trend=true,
此时 main_recommendation 应该是"今天没明显新热点,要不要发你账号最有效的'复盘类'?"

只返回 JSON。`;
```

### 6.2 Account Health — `generateHealthReport(user, tweets, metrics, outcomes)`

```ts
const HEALTH_REPORT_PROMPT = `你是 X 账号分析师。基于用户的推文 metrics + outcomes pattern,生成每周账号健康报告。

## 真实性自审
1. **数据说话** — 报告必须基于输入数据,不编"X 增长靠..."通用话
2. **3 维度** — 报告覆盖:数据(efficiency)/ 趋势(trend)/ 建议(actionable)
3. **建议可执行** — 不是"做更好的内容",而是"你周三发的推文互动比平均低 40%,建议改周二/周四"

## 输入
### 推文 metrics(近 30 天,共 ${tweet_count} 条)
${tweets.map(t => `
- ${t.published_at}: ${t.text.slice(0, 80)}
  类型=${t.inferred_type} 互动=${t.metrics.likes + t.metrics.retweets * 2 + t.metrics.replies * 3}
  impressions=${t.metrics.impressions}
`).join('\n')}

### 关注者增长
start: ${followers.start} → end: ${followers.end} (${followers.delta})

### Outcomes pattern
${outcomes_patterns.map(p => `- ${p.pattern} (confidence: ${p.confidence})`).join('\n')}

### 新关注者样本(${new_followers.length} 人)
${new_followers.slice(0, 10).map(f => `- @${f.handle}: ${f.bio}`).join('\n')}

## 输出 JSON(健康报告)
{
  metrics: {
    followers: { start, end, delta, delta_pct },
    tweets_published: int,
    avg_engagement: float,
    best_time_slot: { day, hour_range, multiplier },
    best_type: { type, avg_engagement, count },
    worst_type: { type, avg_engagement, count },
    new_followers_sample: [...],
    most_quoted: { tweet_text, quote_count, url }
  },
  outcomes_summary: string,                  // 1-2 句话总结 pattern
  recommendations: [string, string, string]  // 3 条 actionable 建议
}

只返回 JSON。`;
```

### 6.3 Teaching Layer — `generateTeachingInsights(user, outcomes, latest_health)`

```ts
const TEACHING_PROMPT = `你是用户账号的"私人增长教练"。基于用户的 outcomes pattern + 最新健康报告,生成 1-2 条"我观察到"的教学 insights。

## 真实性自审
1. **不堆通用建议** — "好内容互动高"不算 insight
2. **必须用户 specific** — "你最近 X 数据显示..." 而不是 "数据显示..."
3. **可执行** — 告诉用户"下次发 X 时,试试 Y"
4. **简短** — 每条 ≤ 2 句话

## 输入
### Outcomes patterns
${outcomes_patterns.map(p => `- ${p.pattern} (${p.confidence})`).join('\n')}

### 最新一周健康数据
${JSON.stringify(latest_health.metrics, null, 2)}

## 输出 JSON
{
  insights: [
    {
      type: 'pattern' | 'tip' | 'warning',
      text: string,                // '你上周 5 条"复盘"贴互动平均 28,其他类型 12'
      evidence: string,            // '统计样本: 5 vs 8 条'
      signature_seed: string       // 用于去重 dismiss: 'pattern:recap_high_engagement'
    }
  ],
  show_count: 1-2
}

优先 warning > pattern > tip。同一用户一周内同 type 不重复。

只返回 JSON。`;
```

## 9. Edge Cases

### Topic Intelligence

| 情况 | 处理 |
|---|---|
| 没新热点 | `no_significant_trend: true`,主推 = "今天没新热点,要不要发你最有效的'复盘类'?" |
| 用户过往写过所有热点 | 推荐"复盘/更新旧文"角度,不是新内容 |
| Watchlist 为空 | 用 web 搜索 + 用户赛道关键词 |
| 用户没配 x_handle | 跳过 cron(避免空跑),提示"配 x_handle 才能生成推荐" |
| Web 搜索全失败 | 跳过 trend,只用 watchlist 推文 + outcomes pattern |
| 推荐 draft 长度 > 280 字符 | 强制截断到 280,提示"已截断,请编辑" |
| 同一用户 1 天多次点"今日推荐" | `force: true` 时重生,否则读缓存 |
| Cron 失败(cron 队列堵塞) | 用户手动触发生效,不卡住 |

### Account Health

| 情况 | 处理 |
|---|---|
| 用户这周没发推 | 报告说"这周没发推,无数据",给 3 条鼓励性建议 |
| 拉 metrics 失败(cookie 失效) | 报告说"部分数据缺失",缺失字段标注,不全跳过 |
| 关注者增长数据拿不到(cookie 限制) | 字段填 null + 标注"该字段需 X API 才能拿" |
| 用户 1 周前刚注册 | 报告说"账号新,样本不足,2 周后再看" |
| 同一周多次生成 | `unique(user_id, week_start)` 约束,只更新 |
| 历史报告的 metrics 已过期 | 报告是不可变快照,不改 |

### Teaching Layer

| 情况 | 处理 |
|---|---|
| 没 outcomes pattern(样本 <10) | 显示"我还没学到你的 pattern,标记 ≥10 条推文后我能给你更准的建议" |
| 用户 dismiss 了所有当前候选 | 显示通用 greeting("hi Zhuanz")不卡 |
| 用户停顿 5 天回来 | 显示最新 1 条 insight(轻提醒) |
| 用户每天开新对话 | 用 `last_message_at` 判断,只在停顿 ≥ 1 天时显示 |
| 同一 insight 提示过但用户没 dismiss | 7 天后再提示(不永久 dismiss) |
| LLM 生成的 insight 太通用 | prompt 加"必须引用具体数据"+ 人工 review 头几条 |

## 10. Acceptance Criteria

### Topic Intelligence

- [ ] 每天 9 点 cron 自动生成推荐(可关)
- [ ] 手动触发"今天发什么"也能用
- [ ] 推荐理由必须引用真实数据(过去推文 + outcomes + voice DNA)
- [ ] 1 主推 + 4 备选齐全
- [ ] "用了/没用" 反馈可记录
- [ ] 真的没热点时,推荐"复盘类"兜底,不强推

### Account Health

- [ ] 每周日 21 点 cron 自动生成周报
- [ ] 报告含 7 个核心 metrics + 3 条建议
- [ ] 历史报告可查(过去 4 周)
- [ ] 数据缺失时优雅降级,不全跳过
- [ ] 建议 actionable,不是"做更好的内容"

### Teaching Layer

- [ ] 新对话开头 / 停顿 1+ 天时显示
- [ ] 不刷屏(每会话 ≤ 1 次)
- [ ] 用户 dismiss 后不再提示该条
- [ ] 7 天后同 type 可再次提示
- [ ] 没 pattern 时显示"数据不足"通用提示

### 性能

- [ ] Topic recommend 端到端 < 12 秒
- [ ] Health report 端到端 < 20 秒(因为拉大量推文)
- [ ] Teaching layer 加载 < 1 秒
- [ ] 缓存:同一用户今日推荐 1 小时内不重复生成

   - [ ] cron 跑 → 看到今日推荐入库
   - [ ] /topics 页面看到主推 + 4 备选

   - [ ] cron 跑 → 周报入库
   - [ ] /health 页面看到报告
   - [ ] 检查 7 个 metrics 字段都填了
   - [ ] 3 条建议是 actionable 的

3. **Teaching Layer 端到端**:
   - [ ] 开新对话 → 看到 teaching banner
   - [ ] dismiss → 重开对话 → 不再显示该条
   - [ ] 1+ 天后再开 → 显示新 insight
   - [ ] 同会话内第 2 条消息 → 不再显示 banner

   - [ ] 模拟 cookie 失效 → health report 标"数据缺失"

### LLM 输出 review

- [ ] Topic reasoning 引用了真实数据(不是"X 在火")
- [ ] Health report 3 条建议是具体行动,不是通用话
- [ ] Teaching insight 1-2 句话,引用具体数字
- [ ] Draft 推文带 voice DNA,不是 LLM 通用话
